CONTRIBUTING

Merci de votre intérêt pour WEBSite.  
Ce document explique comment contribuer simplement et efficacement.

---

🛠️ 1. Créer une issue
Avant toute contribution :
- Ouvrir une issue (feature, bug, task)
- Décrire clairement le besoin
- Lier un milestone si possible
- Ajouter l’issue au Project Board

---

🌿 2. Créer une branche
Nom standard :
- feature/nom-fonction
- bugfix/nom-correctif
- task/nom-tache

Exemple :
`
git checkout -b feature/design-system
`

---

🔀 3. Ouvrir une Pull Request
- Décrire les changements
- Lier l’issue : Fixes #XX
- Vérifier que le build passe
- Attendre la review

---

✔️ 4. Règles de code
- Code simple et lisible
- Pas de dépendances inutiles
- Respect de la structure du projet
- Mise à jour de la documentation si nécessaire

---

🚀 5. Merge
Le merge est effectué lorsque :
- La PR est approuvée
- Les tests sont OK
- Le workflow CI/CD est vert

Merci pour votre contribution.
`
